第13章 拷贝控制

引言

  1. 任何类（Class、struct）都需要定义拷贝控制操作，程序员没有定义的拷贝控制操作，编译器会自动合成（定义）缺失的拷贝控制操作。

  2. 拷贝控制操作主要分为三类：

    1. 同类型的另一个对象来完成某对象的初始化：拷贝构造函数(copy ctor)以及移动构造函数(move ctor)
    2. 用同类型的另一个对象来给某对象赋值：拷贝赋值运算符(copy op=)以及移动赋值运算符(move op=)
    3. 此类型的对象销毁时：析构函数(dtor)

1 copy ctor、copy op=、dtor

1.1 copy ctor

1.1.1 概念与性质

  1. copy ctor：如果一个ctor，它的第一个参数是自身类类型的引用，且任何额外的参数都有默认值，这个ctor就是copy ctor。

  2. 一般形式：Foo(Foo const &rhs);
    2.1 我们可以看到它的第一个参数通常是一个T const &（常量引用）。
    2.2 它通常不应该被声明为explicit。

1.1.2 编译器定义的synthesized copy ctor(合成拷贝函数)

  1. 两个作用
    1.1 阻止copy ctor的调用：Foo(Foo const &rhs) = delete
    1.2 依次将另一个对象中的非static数据成员拷贝到需要初始化的对象。

1.1.3 direct init与copy init

  1. direct init：使用()直接调用构造函数初始化。

  2. copy init：没有直接用()初始化对象，需要把某对象拷贝正在创建的对象中。

  3. copy init可能发生的场景
    3.1 =初始化变量：Foo a = b，b可以是Foo类型，也可以是其它类型，只要能被隐式转换成Foo类型。
    3.2 pass by value：将对象作为实参传递给形参为值类型的函数。

     例如`STL`中使用`insert`或者`push`，容器会对参数进行拷贝初始化。
 使用`emplace`则会进行直接初始化。

    3.3 return by value：函数返回一个值类型的对象。
    3.4 list init：用{...}初始化一个数组中的元素或者一个聚合类。

1.1.4 编译器的优化可能绕过copy ctor

std::string null_book = "9-999-9999-9" 优化为 std::string null_book("9-999-9999-9")

这样从copy init转变为了direct init，调用的是std::string(char const *)，而不是copy ctor。

1.2 copy op=

1.2.1 概念与性质

  1. copy op=：本质上是该类重载的赋值运算符中的一种，只是右侧对象为类本身类型。一般都是该类的成员函数。

  2. 一般形式：Foo &operator=(Foo const &rhs)
    2.1 赋值运算符一般返回指向左侧对象的引用，这样可以完成链式赋值(a = b = c)，copy op=中通过return *this返回左侧对象的引用。

    标准库的容器通常要求存储的元素类型要实现copy op=，并且返回左侧对象的引用。
    2.2 当类需要手动释放资源时（不能使用synthesized copy op=），copy ctor一定要检查自我赋值是否会出问题。

1.2.2 synthesized copy op=

简单的思考

1. 为什么copy ctor的参数必须是引用。通常是T const &。

如果copy ctor的参数不是引用，而是一个值类型，当第一次调用copy ctor时，我们需要将对象作为实参再次调用copy ctor来传递给拷贝构造函数，这会导致copy ctor的循环调用。

2. 为什么copy ctor不应该被声明为explicit。

当我们使用explicit声明一个拷贝构造函数时，我们无法使用copy init(拷贝初始化)来初始化一个对象。只能使用direct init(直接初始化)。

3. synthesized copy ctor的拷贝流程

synthesized copy ctor会拷贝所有的非static数据成员，根据成员类型的不同，采用不同的拷贝方法。

  • 内置类型成员：直接拷贝(int a = b)
  • 类类型成员：调用成员的copy ctor进行拷贝
  • 内置数组类型成员：因为内置数组不能直接拷贝，所以会对数组进行逐元素拷贝。

4. copy op=能够声明为全局函数(global function)吗。
