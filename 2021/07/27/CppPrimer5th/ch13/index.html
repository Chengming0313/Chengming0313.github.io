第13章 拷贝控制

引言

任何类（Class0、struct）都需要定义拷贝控制操作，程序员没有定义的拷贝控制操作，编译器会自动合成（定义）缺失的拷贝控制操作。

拷贝控制操作主要分为三类：

  • 同类型的另一个对象来完成某对象的初始化：拷贝构造函数(copy ctor)以及移动构造函数(move ctor)
  • 用同类型的另一个对象来给某对象赋值：拷贝赋值运算符(copy op=)以及移动赋值运算符(move op=)
  • 此类型的对象销毁时：析构函数(dtor)

  1. 编译器合成的合成拷贝函数
  • 两个作用
    • 阻止copy ctor的调用：Foo(Foo const &rhs) = delete
    • 依次将另一个对象中的非static数据成员拷贝到需要初始化的对象。
  1. 直接初始化与拷贝初始化

  • 直接初始化：使用()直接调用构造函数初始化。

  • 拷贝初始化：没有直接用()初始化对象，需要把某对象拷贝正在创建的对象中。

  • copy init可能发生的场景

    • =初始化变量：Foo a = b，b可以是Foo类型，也可以是其它类型，只要能被隐式转换成Foo类型。
    • pass by value：将对象作为实参传递给形参为值类型的函数。
      例如STL中使用insert或者push，容器会对参数进行拷贝初始化。
      使用emplace则会进行直接初始化。
    • return by value：函数返回一个值类型的对象。
    • list init：用{...}初始化一个数组中的元素或者一个聚合类。

2. 拷贝赋值运算符

  • copy op=：本质上是该类重载的赋值运算符中的一种，只是右侧对象为类本身类型。一般都是该类的成员函数。
  • 一般形式如下：
    • 赋值运算符一般返回指向左侧对象的引用，这样可以完成链式赋值(a = b = c)，copy op=中通过return *this返回左侧对象的引用。
    • 标准库的容器通常要求存储的元素类型要实现copy op=，并且返回左侧对象的引用。
  1. 编译器生成的合成拷贝赋值运算符

  • 两个作用

    • 阻止copy op=的调用：Foo &operator=(Foo const &rhs) = delete
    • 依次将另一个对象中的非static数据成员拷贝给被赋值的对象。

  • 合成拷贝赋值运算符与合成拷贝函数处理过程差不多（逐个初始化与逐个赋值）。

  • 析构函数执行一定是用来销毁某个对象的，记住只要销毁某对象，必然调用它的析构函数

    • delete以及delete[]回收动态内存时，创建的对象也会被销毁。（智能指针本质上也是通过delete销毁对象）
    • 临时变量所在的表达式执行结束时被销毁。

    • 标准库容器以及数组被销毁时，它的成员会销毁。

简单的思考

1. 为什么copy ctor的参数必须是引用。通常是T const &。

如果copy ctor的参数不是引用，而是一个值类型，当第一次调用copy ctor时，我们需要将对象作为实参再次调用copy ctor来传递给拷贝构造函数，这会导致copy ctor的循环调用。

2. 为什么copy ctor不应该被声明为explicit。

当我们使用explicit声明一个拷贝构造函数时，我们无法使用copy init(拷贝初始化)来初始化一个对象。只能使用direct init(直接初始化)。

3. synthesized copy ctor的拷贝流程

synthesized copy ctor会拷贝所有的非static数据成员，根据成员类型的不同，采用不同的拷贝方法。

  • 内置类型成员：直接拷贝(int a = b)
  • 类类型成员：调用成员的copy ctor进行拷贝
  • 内置数组类型成员：因为内置数组不能直接拷贝，所以会对数组进行逐元素拷贝。

5. 析构函数的作用是什么？析构函数体为空，那该类型对象的成员还会销毁吗？

  • 析构函数并不会直接销毁对象的成员变量，它负责执行程序员编写的对象销毁时的附加代码（例如释放分配的动态内存，或者减小引用计数）。

  • 销毁成员变量其实是由析构函数之后执行的析构阶段负责。

  • 一个类对象从创建到销毁的流程：

    1. 构造阶段（初始值列表initialization list）：成员变量的初始化阶段，如果没有使用initialization list，会进行default initialization。
    2. 执行构造函数体
    3. …
    4. 执行析构函数体
    5. 析构阶段：按成员初始化的逆序逐一销毁成员。

6. =default 与 delete 的用法有什么区别。

  • =default只能用于定义类的拷贝控制函数，既可以在类内定义，也可以在类外定义（类内声明），与普通函数一样。

  • =delete可以定义所有的成员函数（主要是拷贝控制函数），用来阻止该成员函数的使用，只能在类内定义。

  • =delete只能在类内定义的理解：=default函数只会影响这个函数的代码，编译时只需要知道函数声明即可，而=delete函数编译时需要告知编译器阻止对它的调用。

7. 编译器生成的拷贝控制函数什么时候是=delete

因为类的拷贝控制操作依赖于成员的拷贝控制操作，所以当某类的数据成员中不能完成拷贝控制操作，该类对应的拷贝控制操作会被定义为=delete。

8. 析构函数被定义为=delete

  • 不能定义该类的变量。
  • 不能创建该类的临时对象。

变量与对象的区别：C++中，大部分情况下变量与对象等价，唯一的区别在于变量一定有变量名，而对象没有名字的概念。

9. copy ctor、copy op=以及dtor的关系

  • 需要自定义dtor时，同时也需要自定义copy ctor以及copy op=

  • copy ctor与copy op=是等价的（要么都自定义，要么都不自定义）

因此考虑某个类的拷贝控制操作时，优先考虑析构函数。

10. private代替=delete

C++11标准之前，通过将某个类的成员函数声明为private的，同时不给出定义来阻止对该函数的调用。

  • 其它成员函数以及友元函数调用该函数，会发生链接时错误。

  • 非成员函数以及友元函数调用该函数，会发生编译时错误。
