1. const 含义

  • const 用来修饰变量，表示变量的值不可改变(赋值), 是一个 只读变量 。
  • const 修饰的变量不一定是一个 常量表达式 , 常量表达式除了要求值不可更改之外, 还要求在编译过程中就能得到计算结果。
  • 一般使用 constexpr 来修饰一个常量表达式, 编译器会对变量进行验证。不符合要求会报错。
  • 声明 constexpr 的类型有所限制, 必须属于 字面值类型 。

2. 定义一个 const 变量

  • const 变量必须初始化
  • const 变量不能赋值
  • const 修饰的 类变量 , 只有在 构造函数 完成后才算真正具有 const 属性。

3. const 的一些特性

  • 用 const 代替 #define
    如下, #define 定义的 SIZE_NUM, 在 预处理过程 就已经被 10 替换, 不会进入 符号表。程序出现问题时不好定位。
  • const 变量默认为文件局部变量

    非 const 变量不同文件访问

const 变量不同文件访问

const 变量需要显式使用 extern 定义, 才能跨文件访问

4. 指针与 const

const 总是优先与它左侧的符号配对

  • 指向常量的指针（底层 const）

    可以不用初始化, 不能通过指针解引用去修改它的值

可以指向非 const 的变量

不能用 void * 指向 const 变量, 必须使用 const void *

  • 常量指针（顶层 const）

    必须初始化, 指针不能修改

error: const T * —-> T * const

5. const 变量的拷贝操作

顶层 const 对象可以拷贝给非顶层 const 对象

非顶层 const 对象也可以拷给顶层 const 对象

底层 const 对象不能拷给非底层 const 对象

非底层 const 对象可以拷给底层 const 对象

6. 函数与 const

6.1 修饰返回值
  • return by const value，一般情况下,，没有意义。一些特殊情况下是有意义的, 详见Effective C++ 条款21。
  • return by const reference

    返回普通引用, 可以修改返回值, 可以作为左值使用

返回常量引用，不能修改

  • return by T const *
  • return by T * const
6.2 修饰参数
  • 普通的 顶层 const (不是 T const &)
    使用普通的 顶层 const 作为函数参数，没有意义, 函数收到的是一个副本。
  • 底层 const
    通过传入 T const * 的方式, 使得函数能够通过指针访问函数体外的对象, 又保证了数据的不可修改。
  • T const &
    pass by const reference 能让函数直接通过变量名访问作用域外的对象, 还能保证数据的 read only

对于类类型的变量来说, 使用pass by value 的方式传值, 在拷贝过程中对象的拷贝构造以及析构都会消耗时间, 效率较低。我们可以使用 pass by pointer to const 以及 pass by const reference 来提高效率, 同时保证数据的安全性。

7. 类与 const
  • const member function

  1. 普通的member function 中的 this pointer 类型为 T *, 而 const member function 中的 this pointer 类型为 T const *。

  2. 对于 const object来说, 它不能用来给普通的this pointer初始化, 它只能给const member function中T const *类型的this pointer初始化。

  3. 对于普通的 object, 它即可以使用const member function, 又可以使用普通的 member function。因为T * 既可以给 T * 初始化, 又可以给 T const * 初始化。

  4. 除了函数参数列表, 是否是 const member function, 也可以进行重载. 普通的 object 默认调用 非 const member function.

  • const 成员

    这里讨论一下类内初始值与初始值列表的优先级。初始值列表的优先级比类内初始值要高。只有没有为该成员设定初始值列表时, 它才会使用类内初始值进行初始化。

类中的 const 成员可以使用类内初始值的方式初始化

类中的 const 成员可以使用初始值列表的方式初始化

不能在构造函数中用赋值表达式初始化 const 成员, 在进入构造函数体之前, 成员的初始化过程已经结束。

当要表示类中共享的常量时, 通常使用 static const 修饰

  1. 采用与一般 static 变量一样的方法, 类内声明, 类外定义
